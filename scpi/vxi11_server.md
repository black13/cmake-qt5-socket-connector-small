# VXI-11 Spectrum Analyzer Server

This repo includes a minimal VXI-11 server that emulates a PSA-class spectrum analyzer over ONC/RPC.
It is intentionally lightweight so we can iterate on SCPI behavior without pulling in external libraries.
The core RPC sources (`vxi11core_*.c`, `vxi11core.h`) are generated by `rpcgen` from `vxi11core.rpcl`.

## Scripting Hook (Duktape)
If `third_party/duktape/duktape.c` + `duktape.h` are present, `scripts/vxi11_build.sh` compiles them and
defines `VXI11_USE_DUKTAPE`. The server will try to load a script from `VXI11_SCRIPT` (or default
`scpi/vxi11_script.js` if that file exists). Set `VXI11_SCRIPT=off` to disable script loading.
Current managed source: Duktape `2.7.0` (see `third_party/duktape/VERSION`, sourced from `duktape-2.7.0.tar.xz`).

The script can implement `handle(cmd)`:
- Return a string to handle the command (a trailing newline is added if missing).
- Return `null`/`undefined` to fall back to the built-in SCPI handler.

Available `vxi` helpers (C <-> JS bridge):
- `vxi.log(level, message)` (level can be number or string: error|warn|info|debug|trace)
- `vxi.getTrace()` / `vxi.setTrace(array)`
- `vxi.getTracePoints()` / `vxi.setTracePoints(points)`
- `vxi.getCenterHz()` / `vxi.setCenterHz(hz)`
- `vxi.getSpanHz()` / `vxi.setSpanHz(hz)`
- `vxi.getToneHz()` / `vxi.setToneHz(hz)` / `vxi.setToneEnabled(bool)`
- `vxi.setNoiseFloorDbm(dbm)` / `vxi.setNoiseVariationDb(db)`
- `vxi.generateTrace()`

Example script: `scpi/vxi11_script.js`.
Example custom commands in that script:
- `:SIM:TRACE:TRI`
- `:SIM:TONE:ON` / `:SIM:TONE:OFF`

Note: Duktape is pure C, so JS arrays are copied into native storage (and vice versa). For complex native state
or STL containers, expose small wrapper functions like the ones above and keep native pointers hidden on JS objects.

## Build (Linux VM)
```
./scripts/vxi11_build.sh
```
Defaults to Debug builds. For Release builds:
```
./scripts/vxi11_build.sh release
```
If the shared folder is read-only, the build falls back to `~/.cache/vxi11_server_debug` (or `_release`).

## Run
```
./scripts/vxi11_server_ctl.sh start
./scripts/vxi11_server_ctl.sh status
```
For debug logging, set `VXI11_DEBUG=1` before starting. Logs go to the runtime log file (see `vxi11_server_ctl.sh`).
To stream logs:
```
./scripts/vxi11_server_ctl.sh logs
```
For deep logging, set `VXI11_LOG_LEVEL` (error|warn|info|debug|trace). You can also force a log file via
`VXI11_LOG_FILE=/path/to/file` (defaults to stderr, which is already captured by the control script).
Log lines include `file:line func()` for C/C++ call sites; JS logs are tagged `JS` and show the C bridge location.

## Debugging with gdb
Run the server under gdb and break on a command (default `*IDN?`):
```
VXI11_BREAK_CMD="*IDN?" ./scripts/vxi11_server_ctl.sh gdb
```
When the command is received, the server raises `SIGTRAP` and gdb stops.

## One-shot VM Debug Run (host)
From the host, use this wrapper to rebuild, restart with the script hook, and run a quick SCPI check:
```
./scripts/vm_vxi11_debug.sh
```
Requires `scripts/vm_config.sh` to be set up and SSH access to the VM.

## Build Cadence
- Default to Debug builds while iterating.
- Do a weekly Release build (`./scripts/vxi11_build.sh release`) followed by the quick SCPI test below.

Ensure `rpcbind` is running before the server starts:
```
sudo systemctl enable --now rpcbind
rpcinfo -p localhost | grep 0607A
```

## Quick Test
```
sudo apt install -y lxi-tools
lxi scpi -a 127.0.0.1 "*IDN?"
```

Expected response:
```
SIMILAR,E4404B,MY00000000,A.01.00
```

- `:SWEep:POINts` / `:SWEep:POINts?`
- `:SWEep:TIME` / `:SWEep:TIME?`
- `:INIT:CONTinuous` / `:INIT:CONTinuous?`
- `:DISPlay:WINDow:TRACe:Y:SCALe:RLEVel` / `:DISPlay:WINDow:TRACe:Y:SCALe:RLEVel?`
- `:INIT` / `:INIT:IMMediate`
- `:TRACe:TYPE` / `:TRACe:TYPE?`

## Provenance Notes
- Vendor payloads (for example, NI-VISA extractions) are only used for local reference and are not kept in git. Recreate them from the official installer if needed.

## Sweep Behavior
The default sweep is tuned for FM-band scans (center 98.5 MHz, span 20 MHz) with a single strong tone injected so
UI workflows can `:CALC:MARK1:MAX` and then center on the peak. A fresh noise floor is generated per sweep so the
trace looks realistic; use `:INIT`/`:INIT:IMMediate` to trigger new noise. The tone also drifts slightly per sweep
(default ±50 kHz with 5 kHz steps) to mimic real-world movement. Adjust `:FREQuency:CENTer`,
`:FREQuency:SPAN`, or the sweep settings to move the tone inside the current band.
